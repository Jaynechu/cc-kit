---
name: grill
description: Interview the plan or design until reaching shared understanding, resolving each branch of the decision tree. Use when user wants to stress-test a plan, get grilled on their design, or mentions "grill".
---

<what-to-do>

Interview yourself relentlessly about every aspect of this plan and make decisions for each branch of the design tree, resolving dependencies between decisions one-by-one.
Make sure we reach a shared understanding.
Do not make-up unecessary questions if there is none.
If you already know there is a better option, just say it.

Optional (depend on how big is the plan): If it's first round grilling or new project, ask the questions one at a time, waiting for feedback on each question before continuing.
  - Grill all together if just single session.

If a question can be answered by exploring the codebase, explore the codebase instead.

</what-to-do>
